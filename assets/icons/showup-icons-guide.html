03 — Setup

How to Add to showup.

1
Pull the latest code — the icons are already in the repo at /assets/icons/ after the last git push.
2
Link the CSS in index.html — add this line after the other CSS links:
<!-- In index.html, add after tracker.css -->
<link rel="stylesheet" href="css/icons.css" />
3
Import the JS helper in any JS file that needs icons:
import { icon, iconEl, iconSrc, STICKER_ICONS } from './icons.js';
04 — Usage

Where to Use Them

Anywhere you'd use an emoji, use an icon instead. Below are the exact spots in the app and the code to use.

Month Bar — Streak Stat

3 day streak · 3/4 days done 
// In app.js updateStat()
import { icon } from './icons.js';

const parts = [];
if (streak > 0) parts.push(`${icon('flame', 16)} ${streak} day streak`);
parts.push(`${doneThisMonth}/${totalThisMonth} days done`);
monthBarStat.innerHTML = parts.join(' · ');
// Note: use .innerHTML instead of .textContent

Theme Toggle Button

dark mode off vs dark mode on 
// In app.js dark mode toggle
import { iconEl } from './icons.js';

// Replace the 🌙/☀️ text content with icon elements
themeToggle.innerHTML = '';
themeToggle.appendChild(iconEl(next === 'dark' ? 'sun' : 'moon', 16));

Month Setup Sticker Picker

// In month-setup.js — replace emoji sticker grid
import { iconEl, STICKER_ICONS } from './icons.js';

// Build sticker picker buttons
STICKER_ICONS.forEach(name => {
  const btn = document.createElement('button');
  btn.className = 'sticker-btn';
  btn.dataset.sticker = name; // store icon name as sticker value
  btn.appendChild(iconEl(name, 24));
  btn.addEventListener('click', () => selectSticker(name));
  stickerGrid.appendChild(btn);
});

// When rendering a badge, use iconSrc for the sticker img
import { iconSrc } from './icons.js';

badge.innerHTML = `
  ${avatarHTML}
  <span class="badge-name">${entry.displayName}</span>
  <img src="${iconSrc(entry.decoration.sticker)}" width="18" height="18" class="showup-icon" />
`;
05 — Notes

A Few Things to Know

Sticker value in Firestore
The decoration.sticker field in Firestore currently stores emoji strings like "🌻". Once you switch to the icon system, store the icon name string instead: "sunflower", "flame", etc. Existing users will need to re-pick their sticker on next monthly setup.
innerHTML vs textContent
When inserting icons via the icon() helper (which returns an HTML string), always use .innerHTML, never .textContent. If you use .textContent it will render the raw HTML string as text.
Adding new icons later
Drop a new .svg file into /assets/icons/, add its name to the ICON_NAMES array in js/icons.js, and it's available everywhere instantly. No other changes needed.
Firebase hosting
The /assets/ folder will be deployed automatically with firebase deploy — no extra config needed since firebase.json already serves the public folder.
