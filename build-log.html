In Progress Following Tab + Privacy System April 2026

A complete social layer built on a "privacy of content, not presence" philosophy. One-way following, five privacy tiers, two-tone signal copy, and a masonry pinned card grid.

Feature What it does Type
Privacy Tiers New Five tiers per user, applied independently to Calendar and Diary: Sharing, Followers, Low key, Ghost, Private. Private cascades both settings. Ghost does not. New
Low Key Cards New Signal copy replaces raw calendar and diary data. Two-tone: calendar copy is data-flavored, diary copy is feeling-flavored. Nine conditions, each with its own copy variant. New
Signal Copy JSON All copy templates live in data/signal-copy.json -- editable without touching JS. Logic stays in following-signals.js. Hardcoded fallback ensures nothing breaks if fetch fails. Refactor
Ghost Cards New Presence without content. Header color kept, moon icon, "Gone quiet for now." text, gold/amber glow border, 0.72 opacity. No calendar, no diary, no month nav. Design
People View Three sections: Pinned (masonry grid, per-card month nav), Showing Up (compact tap-to-expand rows), Crickets (no tracker this month). Unpin / Unfollow / Unpin all actions. New
One-Way Follows Twitter-style following. No mutual follow required. Followers tier means "visible to anyone who follows you" -- not mutual followers. Design
Feed View Stub only. "Feed coming soon." Full implementation planned next. WIP
v0.5 Diary Phase A Early 2026

The full diary feature -- data layer, mobile UI, desktop widget, photo cropper, and overlay. The most complex feature shipped so far.

Feature What it does Type
Diary Data Layer Firestore subcollection per user per date. Firebase Storage for photos. CORS configured via gsutil. Cloud Functions in asia-southeast1. New
Mobile Day Sheet Diary section below the tracker on mobile. Amber dot on calendar days with entries. Tap to open full diary overlay. New
Full Diary Overlay Two-column layout: note on the left, polaroid photo on the right. Opens with opacity + scale spring animation. New
Photo Cropper Interactive drag/pinch/zoom cropper with Fill and Fit toggle. Canvas-based. Known issue: Safari Fit mode blur needs tuning. New
Desktop Notebook Widget Steno/flip-pad card in closed state. Full-screen sheet with scaleX card-flip transition (not rotateY -- avoids double-flip artifact). Compact date strip with mini calendar bottom sheet. New
v0.4 Tracker Core + Cal Card 2025

The foundation of the app -- monthly activity tracking, the calendar card, and the My Log view.

Feature What it does Type
Calendar Card Month grid with activity dots per day. Decoration system: header color, font, sticker, marker shape. Habit chips legend below the grid. New
Activity Logging Tap a day to log activities. Multi-activity support per day. Marks stored in Firestore per user per month. New
Month Setup Modal to define activities for the month. Activities are scoped per month -- changing next month doesn't affect past months. New
My Log View Personal tracker tab. Shows current month card with full calendar, habit chips, and diary section. New
User Decoration Set during onboarding. Header color, font choice (Inter/Caveat/Fredoka/Playfair), sticker emoji, marker shape (circle/square/pill). New
v0.1 Foundation 2025

Auth, onboarding, and the basic shell.

Feature What it does Type
Firebase Auth Google sign-in. Persistent session via onAuthStateChanged. New users routed to setup flow. New
Setup Flow First-time onboarding: display name, decoration choices, first month of activities. New
Tab Shell Three tabs: Mine, Following, All. Bottom nav on mobile, top nav on desktop. Firebase Hosting on showup.jeni.rocks. New
