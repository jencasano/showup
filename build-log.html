Apr 10, 2026 Shipped FO02
Following Feed View

A single-column chronological feed replacing the "Feed coming soon" stub. Fully tier-aware cards with per-zone privacy badges, ghost copy, low key signal copy, activity chips, 7-day strip, and diary zone. Feed spec locked and committed to repo.

Feed spec (FEED_SPEC.md)
Full feed behavior locked: one card per person per day, upgrades/downgrades in real time, four log/diary states, changed message copy for removed log or diary, all five privacy tiers specified for the feed context.
spec
Shared utils module
following-utils.js extracted with getPrivacy() and renderTierBadge() shared between People view and Feed. Eliminates duplicate definitions across files.
feed-card.js
Modular feed card builder. Per-zone tier badges (activity + diary independently). Ghost, Low key, Sharing/Followers branches. Chips with correct activity colors. 7-day activity strip. Pin button with Firestore write.
Ghost copy (seeded)
Ghost calendar and diary copy pools added to signal-copy.json. Four variants each. Selected via pickCopy() seeded by uid + date — same card shows the same copy to all viewers on the same day. Ghost copy is about the absence, not the person. Low key copy is about the person.
Low key / Ghost distinction
Design principle locked: Low key copy uses {firstName} and is warm and observational. Ghost copy names nobody and speaks as the feed itself. This distinction is a named design principle in FEED_SPEC.md.
spec
Changed message copy
Copy variants written for when a log is removed (diary still exists) and when a diary is removed (log still exists). Stored in signal-copy.json. Implementation pending.

Apr 9, 2026 Shipped AL01
All Tab — Discovery Directory

A fully tier-aware discovery directory. Browse everyone showing up, respecting each user's calendar and diary privacy independently. Per-card month nav, four card types, shared diary strip component.

Four card types
Sharing (full calendar + diary strip), Locked (Followers tier, stranger view), Low key (signal copy), Ghost (moon treatment, gold glow). Private users excluded entirely.
spec
Per-card month nav
Each Sharing card browses its own month independently. Global month bar hidden on All tab. Browsing one person's history doesn't affect other cards.
Tier-aware diary strip
Diary zone renders for all visible card types (not just Sharing). Diary privacy is independent of calendar privacy. Ghost diary shows moon treatment. Low key diary shows signal copy. Followers diary respects follow status.
spec
Search
Smart multi-token scoring by name and activities. Sharing tier users searchable by activity name. All other tiers searchable by name only.
Module refactor
mobile-tracker.js (61KB) split into cal-card.js and diary-mobile.js. renderDiaryStrip extracted to shared diary-strip.js used by both Following and All tabs. mobile-tracker.js reduced to 146-byte re-export shim.
Private cascade
If either calendarPrivacy or diaryPrivacy is private, user is excluded from All tab entirely. Enforced at render time as a safety net.

Apr 9, 2026 Shipped FO01
Following Tab + Privacy System

A full social layer built on one principle: privacy of content, not of presence. One-way following, five tiers, two-tone signal copy, ghost cards, and a masonry pinned grid.

Five privacy tiers
Sharing, Followers, Low key, Ghost, Private — set independently for Calendar and Diary. Private cascades both. Ghost is per-zone and doesn't cascade.
spec
Ghost cards
Presence without content. Header color kept, moon icon, "Gone quiet for now." Ghost is per-zone — calendar and diary can be Ghost independently. Gold glow border.
Low key signal copy
Two-tone copy replaces raw data. Calendar copy is data-flavored. Diary copy is feeling-flavored. Nine conditions, 18 copy keys, loaded from signal-copy.json.
People view
Three sections: Pinned (masonry grid, per-card month nav), Showing Up (compact tap-to-expand rows), Crickets (no tracker this month). Unpin / Unfollow / Unpin all with "Sure?" confirm.
Pinned grid (masonry)
CSS column-count:2 with break-inside:avoid. Pin order preserved via Firestore arrayUnion insertion order. L-to-R order backlogged — revisit when CSS Masonry has wider browser support.
Feed view
"Feed coming soon." stub. Full implementation shipped in FO02.

Apr 5, 2026 Shipped DI01
Diary Phase A

The full diary feature — data layer, photo cropper, two-column overlay, pages modal, premium transitions, and the diary notebook widget. The most complex feature shipped so far.

Diary data layer
Firestore subcollection per user per date. Firebase Storage for photos. CORS configured via gsutil in Cloud Shell. Cloud Functions deployed to asia-southeast1.
Photo cropper
Interactive drag/pinch/zoom with Fill and Fit toggle. Canvas-based. JPEG export at 0.82 quality. Known issue: Safari Fit mode blur needs tuning (canvas downscale/upscale workaround in place).
Diary overlay
Two-column layout: note left, polaroid photo right. Blurred backdrop. Esc/outside close with dirty check. Opens with opacity + scale spring animation.
Pages modal
Mini polaroid thumbnails per diary day. Fade transition between days. Page nav moved to bottom footer, coral dot separator.
Transitions
True crossfade between diary modals — no gap frame flicker. Module-level diary entry cache. Sequential stagger index (not day number) for correct timing.
Diary notebook widget
Color themes (coral/cream/indigo) with palette picker. Tilt animation, idle sway, "read it" hint copy. No-reload theme swap, modal spine/corner theming. Amber dot indicator on days with entries.
