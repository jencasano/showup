Polish · Login screen, round 1

First of a series of focused polish passes that follow the Mixtape Phase 2 ship. This round cleans the login screen and its adjacent toasts: theme-aware decorative dots, a voice-corrected sign-out toast, and a hard fix for the ghost error toast that was popping up on the login screen right after sign-out. Docs reorg landed alongside so every design spec now lives under docs/.

Docs reorg
Every design spec moved from repo root into docs/: MIXTAPE_SPEC.md, ROCKSTACK.md, FOLLOWING_SPEC.md, FEED_SPEC.md, FEED_EVENT_SPEC.md, ALL_TAB_SPEC.md, FEED_BUILD.md, DUMMY_DATA.md. Added docs/README.md as the folder index. Reserved docs/polish/ for scoped polish specs. Build-log spec-link hrefs updated, docs/playbook.html appendix links updated. Root now stays focused on infra configs and entry-point HTML.
readme
Login dots, theme-aware
The decorative dots under the Sign in button were pulling from a legacy --color-teal hex (#80B9B9) hardcoded across both themes, so they read as a stale pre-Mixtape color on Side B. Introduced a role-descriptive --accent-support token in css/variables.css that maps to var(--blue). Side B already remaps --blue to #5EAAA8 teal, so a single-line definition auto-resolves per theme. login.css's .login-dot.teal now reads from the new token. Class name kept so no HTML ripple. Legacy --color-teal alias stays in place pending the toast spec sweep.
spec
Sign-out toast, voice-corrected
"Signed out!" violated two Mixtape voice rules: exclamation point on a non-celebratory event, and sentence case on a context (toast) the spec explicitly flags as intentionally-lowercase. Fixed to signed out. lowercase with a trailing period. Same toast type (info), same firing point. Only the string changed.
Logout-race toast fix
Signing out from the Following or All tab was producing a "Failed to load following." (or "Failed to load trackers.") toast on the login screen, because the live Firestore onSnapshot listener was still subscribed when auth flipped to null. Fix is layered: js/app.js sign-out handler now tears down followingUnsub and allLogsUnsub before signOutUser() returns so Firestore has no live listeners at the auth boundary. The tracker-following.js and tracker-all.js error handlers also gate on auth.currentUser; if it's null at error time, we swallow silently. Genuine load failures still surface with voice-corrected copy: couldn't load following. try again? and couldn't load trackers. try again?
What's next
Polish continues. Next up is the monthly setup pass: stickers retuned so they harmonize with whichever badge color the user picks (likely currentColor approach), markers incorporating Mixtape motifs (square / star / squiggle), badge color palette retuned with Mixtape red replacing the legacy coral swatch. Then diary polish (coral accents to Mixtape red), then the diary cover system as a paid-architecture sibling to app themes, then a proper toast spec that retires --color-teal and establishes a full hierarchy of neutral / info / success / error per theme. Scrollbars, empty states with Mixtape flourish, modal audit follow. Icons and landing page stay deferred.

Mixtape · Phase 2 shipped

Mixtape is live in production. showup-jenirocks.web.app flipped from pre-Mixtape to full Side A + Side B in one deploy. Side B: warm leather maroon, salmon red, teal accent, cream text -- a quiet evening companion to Side A's daytime cream. Theme pill, Firestore persistence, per-theme motif SVG, and a cleanup pass on the setup flow (decoration palette, activity rotation, setup.html fonts, setup.css tokens) all shipped alongside.

Side B token layer
css/variables.css got the real Side B palette: paper #2A1518, paper-deep #1D0E10, ink #F0DCCF (cream text), salmon red #F07A5E, teal #5EAAA8, hairline #4A2025. Every consumer-facing token gets overridden inside [data-theme="side-b"] so component CSS keeps reading var(--ink), var(--red), etc. without a rename. Eighteen [data-theme="dark"] selectors across six CSS files flipped to [data-theme="side-b"] so dark-mode overrides activate with the new attribute. --blue remaps to teal on Side B so the stripe auto-flips to red / yellow / teal.
spec
Boot-time flash fixes
Caught during Side B verification: reloading on Side B flashed the cream paper for a frame before js/app.js could catch up. Fixed with an inline pre-paint script in index.html's head that reads localStorage and sets data-theme synchronously, before any CSS paints. Found two more pre-Mixtape rgba leftovers while we were in there: #global-loader was showing old navy indigo on Side B; the month-setup modal backdrop same. Both now use color-mix on --paper / --ink tokens so they auto-theme.
Side A / Side B theme pill
The binary moon/sun toggle is gone. Two-segment pill in the sidebar foot and mobile header, with sun + moon icons (inline SVG, currentColor) inside each segment. Active segment gets the ink fill with paper icon -- the same vocabulary as the sidebar nav active pill. Inline SVG with currentColor instead of the existing sun.svg / moon.svg assets because the existing files hardcode old amber that would clash. Sidebar foot stretches both action rows (Privacy + Sign out, and the pill) to full width so the column has a tidy horizontal rhythm.
Firestore theme persistence
Theme choice follows the user across devices via users/{uid}.theme. A tiny js/theme.js module mirrors the js/diary.js getDiaryTheme / setDiaryTheme pattern. Cold boot still reads localStorage pre-paint; after auth (under the loader), we read the Firestore value and apply it if it differs, or backfill the field if missing. Pill clicks fire-and-forget the Firestore write. Offline writes queue in the SDK and flush on reconnect.
Per-theme motif SVG
The drifting motif layer now swaps palette with the theme. --side-motif-svg custom property on .ambient holds the Side A motif (red, blue, yellow sparkles + dots); [data-theme="side-b"] .ambient overrides with salmon, teal, and yellow. Shape positions, sizes, and opacities identical between themes -- only the three hex values change. Warmth glow and grain stay untouched; they already read fine on both themes.
Setup flow polish
Post-ship audit surfaced six gaps the migration missed in the first-time setup + decoration picker. Decoration color palette in js/month-setup.js swapped from the pre-Mixtape coral / amber / pink / sage set to a Mixtape-harmonious eight-color palette -- the four spec section 4 chromatics (red, blue, yellow, green), both Side B hero colors (salmon, teal), plus cognac and plum as warm/cool bridging tones that echo spec section 11's paid-theme candidates. Activity default rotation in js/utils.js + js/manage-activities.js now uses red / blue / yellow / green / teal straight from the spec. Default decoration fallback #FF6B6B retired in favor of Mixtape red. setup.html's fonts link gained Manrope + DM Mono so the display-name step stops falling back to system sans-serif. css/setup.css dropped a stale @import and swapped its legacy alias tokens for direct Mixtape ones.
What's next
Post-launch backlog: diary theme reharmonization (coral / cream / indigo retuned to the Mixtape palette, or expanded into warm + cool sets per side), the hand-drawn motif art pass that spec section 14 flags as a future exploration, warmth-glow per-theme tuning, state-color tokens for pace badges and tier chips, and the Phase 3 Settings panel + Phase 4 Shop (each gets its own plan).

The Mixtape Redesign · Design locked

A full-session design lock for the showup. redesign. Warm mid-century book meets cassette tape. Side A (cream paper, deep ink, confident red) and Side B (warm leather maroon, salmon red, teal accent) both locked. Mobile M3 variant, brand identity, and theme system architecture all locked. Two canonical artifacts committed: docs/MIXTAPE_SPEC.md as source of truth, docs/playbook.html as visual showcase. Implementation phase (Phase 1: Side A migration) comes next.

Aesthetic direction
Mixtape. Warm 90s and early-2000s nostalgia, not literal. The only mixtape vocabulary that reaches the UI is "Side A" and "Side B" in the theme picker. Everything else is warm editorial restraint. 95% personal / 5% social. Premium-first, monetization-minded.
Side A palette
Paper #F4ECD8, paper-deep #ECE1C6, ink #2A1F1A, red #C3342B (hero), blue #4F6C8E, yellow #E8B33A, green #3E5C3A, hairline #D4C3A1. Derived from six palette studies across sunset, forest, ice cream, and poolside moods.
spec
Side B palette · Warm leather
Paper #2A1518 maroon, text #F0DCCF warm cream, red #F07A5E salmon (tuned for dark bg), teal #5EAAA8 (replaces blue), hairline #4A2025. Sidebar rgba(46,27,26,0.78) with 6px blur. Option 8d won from a three-way final between Option 7 (magenta bruise), Option 8c (teal-tinted shadow), and Option 8d. Stripe: red / yellow / teal.
spec
Level 3 texture
Subtle grain plus drifting warmth gradient (55s ease-in-out) plus drifting brand-color motifs layer (70s linear). Stars, dots, and squiggles in theme-tuned colors. Fully respects prefers-reduced-motion.
Typography
Sora 800 for the logo. Fraunces for display and headings (italic for emphasis). Manrope for body. Caveat for handwriting (diary). DM Mono for numbers, labels, track numbers, metadata. Rejected: Poppins (too SaaS-generic), Instrument Serif (too editorial), Archivo (too poster-loud).
Sidebar (V3 Hybrid 1)
Frosted translucent column. Nav pills with mono track number (01-06) on the left. Active state is a full ink pill with cream text and a red track number. Hand-drawn motif above the logo (placeholder, to be refined). Six nav items: My log, Diary, Following, All, Shop, Settings.
Page ribbon stripe
Red / yellow / [blue on Side A, teal on Side B] stripe across the full top of the page. Reads as the ribbon around a book. The .stripe-wrap parent sits outside the textured frame to avoid being clipped by the texture's overflow boundary.
Mobile layout (M3 Balanced)
Locked from three mobile variants (M1 Quiet / M2 Loud / M3 Balanced) shown at both 390px and 360px widths. Header: Logo · Shop · Theme · Avatar. Bottom nav with four tabs: Mine · Diary · Following · All. Active state is an ink pill with red icon accent (same logic as desktop). Stripe sits above the bottom nav as a 2px top border.
spec
Brand identity
Wordmark: show[up]. with "up" in theme red, period in normal text color. Tagline: "Show up. Period." Standalone mark: red #C3342B square on ink #2A1F1A background, using Sora's square period shape. Universal across themes. Used for favicon (16-32px), app icon (PWA), and social avatars.
spec
Five brand pillars
The tracker is the heartbeat. The diary is the emotional peak. Warm analog, crisp digital. Quiet by default, personality on contact. Specific, not generic.
Tone & voice + hard rules
Five voice traits (warm, specific, quietly observant, a little sarcastic, never performative). Voice calibration table per context. Do's and don'ts with concrete copy pairs. Forbidden patterns documented (no em-dashes, no AI-ish phrasing, no "the user"). Copy review checklist for every string.

Theme system architecture
Every theme is a token swap via data-theme on <html>. Components read tokens, never hardcode hex values. Seven paid theme candidates preserved from exploration as future Shop inventory: Aged tobacco, Night kitchen, Violet night, Magenta bruise, Teal-tinted shadow, Deep teal abyss, Rockstar. Nothing wasted.
Canonical artifacts
docs/MIXTAPE_SPEC.md is the source of truth: 14 sections covering philosophy, tone & voice, hard rules, tokens, motion, components, empty/error states, responsive, mobile, brand, theme system, iconography stub, migration plan, open questions. docs/playbook.html is the visual companion: sticky TOC, live swatches with click-to-copy hex, typography specimens in real fonts, component previews, motion demos, texture with live drift animation.

Apr 15, 2026 Shipped FO04
Feed Event Stream

Feed redesigned from day-card model to event-stream model. Each log or diary write produces one event with tier-specific narrative copy. 10-second debounce batches rapid log taps. Activity names shown inline with smart collapsing. ~120 copy variants across Sharing, Low key, and Ghost tiers.

Event-stream model
Unit is the event, not the day. Each meaningful write (log or diary) generates one feed event. Replaces the old day-card that showed chips, 7-day strips, and diary previews per user per day.
spec
10-second debounce
Activity taps fire Firestore writes individually. feed-debounce.js batches them with a 10-second quiet window -- one event per burst. Diary writes fire immediately (no debounce). First snapshot per user bypasses debounce for instant initial load.
Tier-specific narrative copy
Sharing tier: specific and warm, names activities inline. Low key: insight without activity names. Ghost: real name and avatar shown, but poetic copy that reveals nothing. Copy selected via feed-copy.js with seeded random variants per uid+date.
Three-layer copy styling
Action line has three visual layers: fact (normal text), streak commentary (muted, via {streak} markers), and nudge (coral italic for Sharing, teal italic for Low key, via {nudge} markers). feed-copy.js converts markers to styled spans at fill time.
Activity name collapsing
Sharing tier shows activity names in Fraunces italic 600 inline. Smart format: 1 name, "A and B", "A, B, and C", or "A, B, and N more" for 4+.
Extended context signals
computeSignal now returns contextKey alongside existing headlines. New contexts: streak_full_month (every day this month), backfill (past day, not today), comeback_big/comeback_small. Old keys preserved for People view.
~120 copy variants
signal-copy.json expanded with feed_{tier}_{type}_{context}_{variant} keys. Covers default, streak_3/7/15/25, streak_full_month, first_ever, comeback_small/big, backfill, diary today/past -- across all three visible tiers.
"New posts" pill
Real-time events queue silently when the user is scrolled down. A fixed pill appears showing the count. Tapping merges pending events and scrolls to top. Matches the FB/IG pattern to prevent the feed from shifting while reading.
Back to top button
Global chevron button in the bottom-right corner. Appears when scrolled past 300px, fades in with transition. Works on all tabs. Sits above bottom nav on mobile.

Feed Sort by Last Updated + Reactive Diary

Feed now sorts by real timestamps instead of day number. Both calendar log and diary writes stamp lastUpdated via serverTimestamp(). Diary cache upgraded from one-time fetch to live onSnapshot listeners, edits propagate to followers' feeds in real time.

lastUpdated on log writes
serverTimestamp() added to both save paths (tracker-mylog.js toggleDay, cal-card.js onDayTap). Every day-mark tap now stamps the log doc with an exact server time.
lastUpdated on diary writes
serverTimestamp() added to saveDiaryEntry in diary.js. Diary edits now have precise timestamps for sort ordering.
Timestamp-based feed sort
Replaced maxActiveDay sort (highest day number) with lastSignalMs, compares log.lastUpdated and diary.lastUpdated, most recent wins. Old docs without the field gracefully sort to the bottom. No backfill needed.
Reactive diary cache
Diary entries upgraded from one-time getDoc to persistent onSnapshot listeners (same pattern as logs). When a followed user edits their diary, the feed re-renders and re-sorts in real time. Listeners cleaned up on unfollow and teardown.
Test harness updated
feed-test.html mock log now includes lastUpdated with a toMillis() shim matching Firestore Timestamp shape.
